Like many other blogging platforms, Nature Network uses Textile (https://web.archive.org/web/20120611040405/http://textism.com/tools/textile/?) to format blog entries. Textile is a markup language, and Andrew Sun has nicely put together the most important tags in this blog post (https://web.archive.org/web/20120611040405/http://network.nature.com/blogs/user/andrewsun/2007/09/09/textism-a-better-guide-to-formatting-in-nature-network?).

Unfortunately, Textile doesn't much help with formatting that is specific to science blogging. Specifically, there is no standard way to link to journal articles. Should we use the DOI (https://web.archive.org/web/20120611040405/http://www.doi.org/?), should we link to Pubmed (https://web.archive.org/web/20120611040405/http://www.pubmed.gov/?), or should we link to the journal homepage? If we do the latter, should we link to the fulltext (that might not be accessible to everybody) or the abstract? And why is there no standard markup for this?

MediaWiki (https://web.archive.org/web/20120611040405/http://www.mediawiki.org/?), the platform behind Wikipedia (https://web.archive.org/web/20120611040405/http://www.wikipedia.org/?), uses a different markup language. Here, linking to PubMed (https://web.archive.org/web/20120611040405/http://en.wikipedia.org/wiki/Wikipedia:PMID?) is semiautomated. Linking to DOIs (https://web.archive.org/web/20120611040405/http://de.wikipedia.org/wiki/Vorlage:DOI?) is also supported. And the Biblio (https://web.archive.org/web/20120611040405/http://openwetware.org/wiki/Biblio?) extension, also used by OpenWetWare (https://web.archive.org/web/20120611040405/http://openwetware.org/?), makes linking to papers even easier.

We have often talked about standard data formats. And the small things also matter. One result of standard blog linking to papers: it would become much easier to find all blog posts that link to a particular paper.
